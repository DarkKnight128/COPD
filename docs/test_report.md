# COPD MVP 测试报告

## 测试范围

本轮测试覆盖第 11-12 周 MVP 验收交付功能：

- 登录、退出和角色权限
- 数据导入和导入日志
- 患者列表检索、筛选、删除
- 患者详情和病程时间轴
- 本地规则评估和 API 智能评估入口
- 模型调用日志和 LangGraph 节点日志
- 关键证据追踪
- 报告生成、编辑、版本管理、确认、驳回、导出
- 操作日志查询
- 安全边界检查

## 自动化测试命令

```powershell
$env:QWEN_ENABLE='false'
python -m unittest discover -s tests -v
```

## 重点用例

| 测试项 | 预期结果 |
| --- | --- |
| 未登录访问患者页 | 跳转登录页 |
| 未登录访问 API | 返回 401 |
| 科研人员访问患者列表 | 成功，只读 |
| 科研人员触发评估 | 返回 403 |
| 管理员导入 100 例模板 | 导入成功，患者数量正确 |
| 管理员删除患者 | 患者及关联记录被删除 |
| 医生触发本地规则评估 | 生成评估结果、模型日志和报告 |
| 医生编辑报告 | 版本号递增，状态回到待复核 |
| 医生确认报告 | 报告状态为已确认 |
| 医生驳回报告且原因为空 | 返回 400 |
| 管理员查看操作日志 | 能看到关键动作记录 |
| 页面/API 日志泄露 API Key | 不应出现 |

## 安全边界

测试报告正文和导出页不应包含：

- 具体治疗方案
- 处方建议
- 替代医生最终诊断的表述
- `.env` 或 `DASHSCOPE_API_KEY`

## 当前结论

系统已具备单中心试用或项目组验收演示所需的主流程。真实 HIS/EMR/PACS、原始 DICOM、复杂账号体系、多中心部署和生产级性能监控仍属于后续版本。
