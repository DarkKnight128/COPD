# COPD MVP 验收演示脚本

## 演示目标

展示系统可以稳定完成：

```text
登录
-> 导入 100 例样例数据
-> 筛选高风险患者
-> 查看患者总览和病程时间轴
-> 触发智能评估
-> 查看模型版本和关键证据
-> 医生编辑、确认或驳回报告
-> 导出基础版慢阻肺智能辅助评估报告
-> 管理员查看操作日志
```

## 演示步骤

1. 启动系统

```powershell
python scripts\run_poc_server.py
```

2. 浏览器打开

```text
http://127.0.0.1:8001
```

3. 管理员登录

- 账号：`admin`
- 密码：`admin123`

4. 导入数据

- 点击“导入”。
- 上传 `data/copd_patient_import_sample_100.xlsx`。
- 展示导入成功统计和导入批次。

5. 查看患者列表

- 返回“患者列表”。
- 按风险等级筛选“高”。
- 展示评估状态、复核状态、报告状态、随访状态和导入批次。

6. 医生登录

- 退出管理员。
- 登录 `doctor / doctor123`。

7. 进入高风险患者详情

- 查看基础信息、症状评分、肺功能、CT 摘要、用药、急性加重和随访。
- 点击“查看病程时间轴”，展示多类型事件按时间排序。

8. 触发智能评估

- 点击“本地规则评估”用于稳定演示。
- 如 `.env` 已配置并确认可用，可点击“API 智能评估”。

9. 展示评估结果

- 数据质量
- 表型与风险
- 模型调用与版本
- 输入数据版本
- 关键证据来源、日期和字段
- 安全检查
- LangGraph 节点日志

10. 医生复核报告

- 点击“编辑报告”。
- 修改报告正文。
- 填写复核意见。
- 点击“确认报告”或“驳回报告”。
- 展示版本记录和医生操作记录。

11. 导出报告

- 点击“导出报告”。
- 点击“打印 / 导出 PDF”。
- 强调报告包含患者摘要、评估结果、关键证据、医生意见、版本号和安全声明。

12. 管理员查看操作日志

- 退出医生，登录管理员。
- 点击“操作日志”。
- 展示登录、导入、评估、报告编辑、确认/驳回、导出等动作。

13. 管理员查看系统配置

- 点击“系统配置”。
- 展示数据库路径、导入模板路径、100 例样例数据路径、Qwen 启用状态和 API Key 是否已配置。
- 强调页面不显示 API Key 明文。

## 演示边界说明

- 本系统是 MVP 验收演示，不是生产医院系统。
- 不接 HIS、EMR、PACS。
- 不处理原始 DICOM。
- 不输出具体治疗方案。
- 医生复核结果保留为后续模型迭代和临床反馈闭环的数据基础。
